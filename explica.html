01 — Visão Geral

O que é o sistema

Toda magia é construída em camadas. A base é o Núcleo, que define qual elemento existe. Sobre ele empilham-se Kernels (propriedades físicas), Aditivos (modificadores), Conectivos (operadores lógicos) e Subciclos (blocos reutilizáveis). O compilador percorre tudo isso e produz um resultado ou um colapso.

Hierarquia

Núcleo → Kernels → Aditivos → Subciclos → Camadas → Resultado. Cada nível depende do anterior.

Execução

Lido de dentro para fora. O subciclo mais interno é resolvido primeiro, depois propagado para as camadas externas.

Grafo

Tudo em {} é um grafo completo. As dependências formam um DAG direcionado sem ciclos.

Tipos

Conectivos são tipados. eco -- aumento é inválido. O compilador rejeita antes de executar.

03 — Dimensões

Os Kernels K

Um Kernel é uma dimensão mensurável dentro de um núcleo. Ele funciona como uma variável de estado — tem um valor padrão e pode ser modificado por aditivos. Na notação K_elemento indicamos o kernel de um elemento específico.

04 — Transformações

Os Aditivos A

Aditivos são funções de transformação que modificam os valores de um kernel dentro do vetor do núcleo. Cada aditivo opera sobre uma dimensão específica e produz um novo vetor.

Aumento

A_aumento(K_i) = K_i + Δ
Δ = quantidade fixa por nível
// Aumentar entropia do fogo
A_aum(K_fogo[entropia]) = entropia + Δ

Redução

A_redução(K_i) = K_i − Δ
Δ = quantidade fixa por nível
// Reduzir calor de uma área
A_red(K_fogo[entropia]) = entropia − Δ

Regras de tipo — nem todo aditivo opera em todo kernel

ExpressãoVálido?Motivo
entropia -- aumento✓ simentropia ∈ tipo escalável
força -- aumento✓ simforça ∈ tipo escalável
eco -- aumento✗ nãoeco ∉ tipo escalável → colapso
controle -- aumento✗ nãocontrole ∉ tipo escalável → colapso
aumento -- ponto -- ponto -- ponto✓ simaumenta a área do efeito
05 — Álgebra das Camadas

Camadas C e o Resultado Final

Uma magia completa é construída empilhando camadas sobre o núcleo. Cada camada Cᵢ é um conjunto de aditivos, kernels modificados e subciclos. O resultado final é obtido somando tudo.

// Uma camada
Cᵢ = N + A₁ + A₂ + ... + Aₖ   (k aditivos)
// Resultado final: núcleo + soma de todas as camadas
R = N + Σi=1n Cᵢ

Exemplo — Bola de Fogo

// Código arcano:
[Fogo ==c (ponto--ponto)] -- ponto ({ponto--ponto--ponto} -- aum⁴) -- (gatilho--ponto) ({KFogo} -- aum⁴) (KTerra)esfera

// Decomposição em camadas:
N = N_fogo = [entropia_amb, volume_2cm³, estado_plasma, ...]
C₁ = N + A_ponto + A_ponto       // define o "gatilho de lançamento"
C₂ = N + A_aum + A_aum + A_aum + A_aum  // área × 4
C₃ = K_fogo + A_aum×4              // entropia ×4 → dano alto
C₄ = K_terra → morfologia: esfera   // forma final

R = N + C₁ + C₂ + C₃ + C₄

Leitura: Cada aditivo aum aplicado a entropia acrescenta +Δ ao vetor. Quatro aumentos = entropia + 4Δ. A camada C₃ sozinha é responsável pelo alto dano de fogo da bola de fogo.

06 — Recursão

Subciclos ( )

Subciclos são blocos de escopo — análogos a funções ou parênteses matemáticos. Eles são resolvidos antes do ciclo externo, de dentro para fora. O resultado de um subciclo se torna um aditivo para a camada superior.

// Resolução recursiva: subciclo mais interno primeiro
resolve( A₁ -- A₂ ) → resultado_parcial
ciclo_externo + resultado_parcial → resultado_final
// Exemplo: mover algo com a voz
({ponto -- ponto} -- vocal) -- estado

// Ordem de execução:
1. resolve (ponto -- ponto) → "dois pontos de ancoragem"
2. resolve resultado + vocal → "controle por voz sobre dois pontos"
3. aplica -- estado → "move o estado do objeto entre os pontos"

Ciclo Primitivo

Um subciclo que contém um triângulo isósceles interno. É a forma mais simples e estável de subciclo. Exige mínimo 3 componentes.

Regra do Triângulo

Uma magia com menos de 2 componentes (aditivos ou subnúcleos) é instável. O grafo precisa fechar — mínimo 3 nós para conectividade.

07 — Grafos

Conectivos e Grafos

Os conectivos são os operadores de relação entre nós do grafo mágico. Cada conectivo define o tipo de aresta no grafo dirigido da magia.

Estrutura de grafo — tudo em { } é grafo completo

// {A -- B -- C} = grafo completo: A↔B, B↔C, A↔C
// Formalmente: ∀ nós u,v ∈ {}: existe aresta u→v

{ponto -- ponto -- ponto}
// = triângulo completo — 3 nós, 3 arestas — geometria mínima válida

Fogo ==c (ponto -- ponto)
// = atribui a característica "dois pontos" ao núcleo fogo
08 — Compilador

O que o Compilador faz

O compilador executa seis verificações obrigatórias em ordem. Se qualquer etapa falhar, a magia não executa e o resultado é um Colapso Arcano.

1
SINTAXE
Os tokens estão na ordem correta? Os parênteses fecham? Os símbolos existem?
2
SEMÂNTICA
Os conectivos fazem sentido? Os tipos são compatíveis? eco -- aumento é rejeitado aqui.
3
TOPOLOGIA
O grafo é válido? A magia tem mínimo 3 nós (triângulo)? Todos os nós em { } estão conectados?
4
RESOLUÇÃO MATEMÁTICA
Calcula R = N + ΣCᵢ. Aplica todas as transformações de kernel. Infere dano, área, custo, duração.
5
ESTABILIDADE
A mana do conjurador suporta o custo? A magia não entra em loop infinito? O resultado é fisicamente coerente?
✓
EXECUÇÃO
Todas as verificações passaram → magia conjurada com sucesso.
09 — Tratamento de Erro

Colapso Arcano (1d10)

Quando o compilador detecta um estado inválido em execução, o feitiço falha e produz um Colapso Arcano. O d10 determina a severidade — análogo a níveis de exceção.

1
CATASTRÓFICO — Singularidade Dimensional
Implosão do núcleo → buraco negro temporário. 9m de raio, dano massivo. Inconsciente + Exaustão.
2
CRÍTICO — Efeito Bumerangue Absoluto
Vetor colapsa para dentro. O próprio conjurador absorve dobro do dano original.
3
GRAVE — Fenda Conjuracional
Rasga o tecido da realidade. Um elemental hostil do elemento usado surge e ataca imediatamente.
5–6
MÉDIO — Choque / Exaustão de Afinidade
Dano psíquico ou perda de acesso ao núcleo usado até descanso longo.
7–9
LEVE / INCÔMODO — Estigma, Eco ou Sujeira Arcana
Vulnerabilidade temporária, armadilha residual ou penalidade social/furtividade.
10
INOFENSIVO — Dissipação Fria
A magia simplesmente falha. Recurso gasto, mas nenhum dano. Análogo a um warning.
10 — Resumo

O Sistema Completo

N

Núcleo

Vetor coluna. Cada linha é um kernel. Estado inicial do elemento.

N = [k₁, k₂, ..., kₙ]ᵀ
K

Kernel

Dimensão física dentro do núcleo: entropia, volume, morfologia, velocidade, força…

K_fogo = N_fogo[entropia]
A

Aditivo

Função de transformação sobre um kernel. Muda o valor de uma dimensão do vetor.

A_aum(Kᵢ) = Kᵢ + Δ
C

Camada / Subciclo

Conjunto de aditivos aplicados sobre o núcleo. Resolvidos de dentro para fora.

Cᵢ = N + A₁ + A₂ + ... + Aₖ
R

Resultado

Soma de todas as camadas sobre o núcleo. O compilador infere dano, área, custo, duração.

R = N + Σᵢ Cᵢ
!
